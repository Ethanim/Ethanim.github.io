Linux 下搭建 Java 环境、Tomcat 服务器，MySQL 服务器

发表于 2018-04-16 | 分类于 Linux

上一篇文章演示了在虚拟机中安装 CentOS7，下面介绍如何在 Linux 下搭建 Java 环境、Tomcat 服务器，MySQL 服务器

配置 Java 环境

采用 SecureCRT 连接虚拟机，采用这款软件连接，可以粘贴复制命令，比较方便，前提虚拟机得开启

执行如下命令，进入 /opt 目录，创建并进入 java7 目录

cd /opt
mkdir java7
cd java7

采用 WinSCP 软件连接到虚拟机，这个软件可以上传本地文件到虚拟机里

上传 Linux 版的 jdk 到虚拟机 /opt/java7 目录上，并设置全部权限

执行 ll 命令，即可看到刚上传的 jdk 压缩包

执行如下命令，即可解压该压缩包到当前文件夹，再次 ll 命令，可以看到解压后的文件

tar -zvxf jdk-7u79-linux-x64.tar.gz

执行 vi ~/.bash_profile命令，进入配置环境变量文件

在其中，添加如下配置，其它的不要动

export JAVA_HOME=/opt/java7/jdk1.7.0_79
export CLASSPATH=$JAVA_HOME/jre/lib/rt.jar:$JAVA_HOME/lib/dt.jar:$JAVA_HOME/lib/tools.jar
export PATH=$PATH:$JAVA_HOME/bin


执行source ~/.bash_profile，使配置文件生效，执行java -version，查看 jdk 版本

到此，Linux 下就搭建好了 Java 环境

安装 tomcat

开启虚拟机，用 WinSCP 上传 tomcat 到 Linux 上

用 SecureCRT 工具连接 Linux，进入 /opt 目录，并执行如下命令，给 tomcat 设置全部权限

cd /opt
chmod -R 777 apache-tomcat-7.0.67/

如图，可以看到读写执行权限开启


进入到 tomcat 的 bin 目录，执行如下命令，开启 tomcat 服务器

./startup.sh

查看是否启动成功：
方法一：执行如下命令，进入 logs 文件夹下，查看日志信息，如无报错，并有启动时长，即代表成功

cd ..
cd logs
cat catalina.out

方法二：执行如下命令，查看 tomcat 进程信息，9682 即为进程号，注意：进程号不固定，下次开启会换成别的

ps -ef|grep tomcat

停止 tomcat 也有两种方式：
方法一：切换到 bin 目录下，执行如下命令，关闭 tomcat 服务器

./shutdown.sh

方法二：使用 kill 命令，-9 代表强制关闭；-15 代表，让程序自己关闭，程序可能还会运行一会，会保存当前状态之类的，推荐使用 -15

kill -15 9867
kill -9 9867

再次执行 ps -ef|grep tomcat 命令，看到如下，就代表关闭成功

到此，就介绍了 tomcat 在 Linux 上的配置、开启与关闭。

安装 mysql

进入 /opt 目录下，输入如下命令，创建 mysql57 文件夹

mkdir mysql57

进入 /opt/mysql57 文件夹下，执行如下命令，下载 mysql

wget http://repo.mysql.com/mysql57-community-release-el7-11.noarch.rpm

此时会提示，未找到命令，执行 yum install wget -y，在线安装 wget 命令，如下图就安装成功

再次执行如下命令，在线下载 mysql

wget http://repo.mysql.com/mysql57-community-release-el7-11.noarch.rpm

再执行如下命令

sudo rpm -ivh mysql57-community-release-el7-11.noarch.rpm

再执行如下命令，安装 mysql，会提示如下信息，直接按 y 确认即可，共需要按两次 y

sudo yum install mysql-server



提示下图，就代表成功了

接下来配置 mysql 的相关信息,执行如下命令，登录 mysql，会出现如下错误，这是因为没有权限

mysql -u root

执行如下命令，修改权限

chmod -R 777 /var/lib/mysql

执行如下命令，重启 mysql

service mysqld restart

再次登录，执行如下命令，通过 WinSCP 工具找到密码所在文件，/var/log/mysqld.log 里，如下图，密码为：9fLGGtk7Pl>v

mysql -u root -p

接下来执行如下命令，修改初始密码，会提示密码不符合当前安全策略

alter user user() identified by 'enoch.site';

执行如下命令，修改密码安全策略，再次修改密码

set global validate_password_policy=0;
alter user user() identified by 'enoch.site';

执行如下命令，退出，重启 mysql 数据库，用新密码登录

exit;
service mysqld restart #退出 mysql后执行的
mysql -u root -p

这样就修改好了密码，在本地用数据库连接工具连接 Linux 上的 mysql

会报 2003 错误，是因为防火墙给拦截了，下面的命令，设置防火墙允许 3306 端口可以进出

/sbin/iptables -I INPUT -p tcp --dport 3306 -j ACCEPT

再次连接，会报 1130 错误，原因是默认只能 localhost 访问，修改为 host=’%’，即所有地址都能访问，在给 root 用户设置权限，即可

mysql -u root -p
use mysql;
#修改 访问地址为 %（% 代表任意地址）
update user set host='%' where user ='root' and host='localhost';
select host,user from user;
set global validate_password_policy=0;
#给 root 用户，密码为 enoch.site 的用户授予所有权限
grant all privileges on *.* to root@'%' identified by 'enoch.site' with grant option;

退出，重启 mysql 服务器 service mysqld restart;，再次测试连接，即可成功

到此，就在 linux 上配置好了 mysql 服务器
