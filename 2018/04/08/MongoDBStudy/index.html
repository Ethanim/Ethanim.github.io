MongoDB 学习笔记

发表于 2018-04-08 | 分类于 MongoDB

MongoDB简介

MongoDB 是一个基于分布式文件存储的数据库。
MongoDB 是一个介于关系数据库和非关系数据库之间的产品，是非关系数据库当中功能最丰富，最像关系数据库的。

MongoDB 的特点

  • 支持查询与索引
  • 存储结构灵活，适应多变的数据
  • 运行速度更快
    相比 MySQL、Oracle
  • 存储的数据更多
    MySQL数据库存储的数据量：1000 万以内，数据库性能都可以，上亿条则数据库性能有降低
    Oracle：几亿—-几十亿 ，数据库性能有保障
    MongoDB 单表存储的数据可以是 PB 级的

MongoDB 适合存储的信息

适合存储“大数据量、低价值”的数据
例如：
商品的评论信息
微博系统的 点赞、帖子回复 等等信息

下载与安装

1.下载地址 https://www.mongodb.com/download-center
2.解压后，bin 目录下 mongod.exe 服务器端，mongo.exe 客户端
3.配置环境变量 D:\learn\mongodb\bin
4.创建 data 与 logs 文件夹用户存储数据与日志文件
5.在 bin 目录下执行如下语句，安装服务，--dbpath指定文件存储目录，--logpath指定日志存储位置

D:\learn\mongodb\bin\mongod.exe --dbpath D:\learn\mongodb\d
ata --logpath D:\learn\mongodb\logs\db.log --install

6.在服务中可以看到安装成功，并启动服务

7.如下图，执行 mongo.exe 可以看到启动成功与对应信息，端口号：27017

MongoDB 基本操作

MySQL 与 MongoDB 基本概念对应

MySQL MongoDB
数据库(database) 数据库(database)
表(table) 集合(collection)
记录 文档

库操作

show databases //查看有哪些库

MongoDB Enterprise > show databases
admin  0.000GB
local  0.000GB

use databaseName //创建一个库进入或直接进入库，库里没有集合，则 show databases 不显示

MongoDB Enterprise > use javaDB
switched to db javaDB

db.dropDatabase() //删除当前库

MongoDB Enterprise > db.dropDatabase()
{ "ok" : 1 }

添加操作

db.collectionName.insert(json字符串) //添加 json 数据到 collectionName 集合，第一次添加自动创建该集合

MongoDB Enterprise > db.emp.insert({"no":1,"name":"tom","age":18,"deptNo":10})
WriteResult({ "nInserted" : 1 })
MongoDB Enterprise > db.emp.insert({"no":2,"name":"jack","age":20,"deptNo":20})
WriteResult({ "nInserted" : 1 })

MongoDB 内置了 JavaScript 引擎，可以通过 JavaScript 代码来批量插入

MongoDB Enterprise > for(var i=1; i<=100; i++){
... db.emp.insert({"no":i,"name":"bob"+i,"age":18,"deptNo":20+i});
... };
WriteResult({ "nInserted" : 1 })

查询操作

db.collectionName.find() //查询所有，_id 为自动生成的唯一 id
db.collectionName.findOne() //db.collectionName.findOne() //查询第一条数据

MongoDB Enterprise > db.emp.find()
{ "_id" : ObjectId("5aca1a45b12b5b0373b45bf8"), "no" : 1, "name" : "tom", "age"
: 18, "deptNo" : 10 }
{ "_id" : ObjectId("5aca1a53b12b5b0373b45bf9"), "no" : 2, "name" : "jack", "age"
 : 20, "deptNo" : 20 }

db.collectionName.find(json字符串) //按条件查询

MongoDB Enterprise > db.emp.find({"no":1})
{ "_id" : ObjectId("5aca1a45b12b5b0373b45bf8"), "no" : 1, "name" : "tom", "age"
: 18, "deptNo" : 10 }

非等值查询关键字：”$gt”，大于，”$gte”，大于等于，”$lt”，小于，”$lte”，小于等于，”$ne”，不等于
db.emp.find({"no":{$gt:98}}) //查询no>98的

MongoDB Enterprise > db.emp.find({"no":{$gt:98}}))
{ "_id" : ObjectId("5aca1eb4b12b5b0373b45c5c"), "no" : 99, "name" : "bob99", "ag
e" : 18, "deptNo" : 119 }
{ "_id" : ObjectId("5aca1eb4b12b5b0373b45c5d"), "no" : 100, "name" : "bob100", "
age" : 18, "deptNo" : 120 }

更新操作

db.collectionName.update(条件,更新值) //全部更新

MongoDB Enterprise > db.emp.insert({"no":1,"name":"tom","age":20})
WriteResult({ "nInserted" : 1 })
MongoDB Enterprise > db.emp.update({"no":1},{"name":"jack"});
WriteResult({ "nMatched" : 1, "nUpserted" : 0, "nModified" : 1 })
MongoDB Enterprise > db.emp.find()
{ "_id" : ObjectId("5aca2265b12b5b0373b45c5f"), "name" : "jack" }

先清空表(命令在后面介绍)，添加一条数据，修改后，会发现值只剩修改后的值了，这是因为默认是全部更新
有时候我们仅仅需要更新一个字段，而不是整体更新，那么就可以利用 MongoDB 中提供的修改器
$inc 修改器，自增 $inc 指定的值，如果没有此 key，则会创建 key
$set 修改器，指定更新的字段与对应的值

db.collectionName.update(条件,{$set:更新值}) //局部更新

MongoDB Enterprise > db.emp.insert({"no":1,"name":"tom","age":20})
WriteResult({ "nInserted" : 1 })
MongoDB Enterprise > db.emp.update({"no":1},{$set:{"name":"tom"}})
WriteResult({ "nMatched" : 1, "nUpserted" : 0, "nModified" : 0 })
MongoDB Enterprise > db.emp.find()
{ "_id" : ObjectId("5aca2587b12b5b0373b45c64"), "no" : 1, "name" : "tom", "age"
: 20 }

如果查询条件没有，则可以通过设置第三个参数为 true，来插入数据

MongoDB Enterprise > db.emp.update({"id":100},{"name":"tom"},true)
WriteResult({
        "nMatched" : 0,
        "nUpserted" : 1,
        "nModified" : 0,
        "_id" : ObjectId("5aca28f817a8cf394053a115")
})

在 MongoDB 中如果匹配多条，默认至修改第一条，如需批量修改，把第四个参数指定为 true 即可，就不举例了。

删除操作

db.collectionName.remove({}) //删除所有记录
db.collectionName.remove(json字符串) //删除指定条件记录

MongoDB Enterprise > db.emp.remove({});
WriteResult({ "nRemoved" : 1 })

其它操作

统计数量
db.collectionName.count() //统计该集合的文档数量
db.collectionName.count(json字符串) //按指定条件统计数量

MongoDB Enterprise > db.emp.count();
3

删除集合
db.collectionName.drop()

MongoDB Enterprise > db.emp.drop();
true

查看有哪些集合
show collections

客户端提供了一套帮助系统，通过帮助就可以完成很多任务
系统级别：help
数据库级别：db.help()
集合级别：db.collectionName.help()
方法级别：查看一个方法有几个参数：db.collectionName.funName，如:db.emp.find
