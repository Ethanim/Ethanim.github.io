GitHub学习笔记

发表于 2018-03-22 | 分类于 Git

注册并登录Github

Sign up 注册，然后在，Sign in 登录。

创建项目与文件

Start a project 创建一个项目

这个项目，在GitHub中，叫做仓库（repository），点击 Create repository 就创建好了一个项目

然后就可以创建文件，并提交，这样就创建好了一个文件

桌面版的下载与使用

https://desktop.github.com 可以下载桌面版的GitHub

安装并打开，可以看到三个按钮，分别代表

  • create 没有项目，创建一个项目
  • add 本地已经有项目了，输入路径
  • clone GitHub 上已有项目clone（下载）到本地

    点击Create a new repository 可以在本地创建一个新的仓库

    可以在本地仓库对应文件夹创建文件，蓝色代表选中，绿色代表未选中

    可以对选中的进行提交，就会多一个版本

    选中绿色的部分，可以第二次提交，也可以在History这看到以前的版本，回退到以前的版本

    Publish repository 可以提交到远程仓库（GitHub）上

    填上仓库名与描述，去掉私有的勾，就可以提交到远程仓库（GitHub）上了

    在网页版GitHib中即可看到刚刚提交的仓库

    master分支与创建分支

    master分支是默认的分支，是放程序的地方，当有一个新的想法，但不想修改原来的代码，就可以创建一个分支，可以在新分支上进行修改，但不影响原来的内容

    这里创建了一个 idea 分支


    可以在 idea 分支创建文件，并提交，不会对master分支有影响

    点击可以切换回master分支，可以看到在idea分支创建的文件没有了

    也可以删除分支，默认分支是删除不了的

    在网页端可以查看有哪些分支，也可以创建分支，在Settings里，也可以设置默认分支

    合并分支

    在次创建idea分支，并在idea分支中新建文本，当觉得idea分支的内容比较有用时，就可以合并分支，把idea分支中的内容合并到master分支中


    master分支中就有了idea分支中的内容

    如果两个人都修改同一个文件，修改地方不同，可以提交上
    但是，如果修改同一个地方，就会产生冲突，可以查看冲突，删除冲突的标识符来解决。
