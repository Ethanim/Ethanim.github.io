模版方法设计模式（Template Method）

发表于 2018-03-23 | 分类于 设计模式

什么是模版方法

模版方法模式定义了一个操作中的算法骨架，骨架之中定义了一些算法的执行步骤，但是这个框架会将一些步骤延迟到子类中实现，这样一个子类可以不改变算法结构的同时，可以重新定义算法的某些步骤。

生活中的模版

银行办理业务

在生活中，银行就给客户留下了一个模版。
如下面4个步骤，其中1、3、4较为固定，2就留给每一个顾客来具体填写。
1.进门取号
2.填写单据
3.等待叫号
4.窗口办理

奥运会开幕式

国际奥委会为每一个举办国都定下一个举办步骤，但是在其中，3、5可以由举办国自由发挥，这也是模版。
1.升国旗奏国歌
2.致辞讲话
3.文艺汇演
4.入场仪式
5.点燃奥运圣火

饮料的调制方法

咖啡和茶的泡制也是一种模版方法，1、4步骤相对固定，而2、3步骤就需要用到不同的方法了。

咖啡的泡法 茶的泡法
1.把水煮沸 1.把水煮沸
2.用沸水冲泡咖啡 2.用沸水浸泡茶叶
3.把咖啡倒进杯子 3.把茶叶倒进杯子
4.加糖和牛奶 4.加柠檬

提神饮料的配置

提神饮料泡法
(1)把水煮沸(boilWater)
(2)泡饮料(brew)
(3)把饮料倒进杯子(pourInCup)
(4)加调味料(addCondiments)

如何实现模版方法模式

实现要素
1.准备一个抽象类，将部分逻辑以具体方法的形式实现，然后声明一些抽象方法交由子类实现剩余逻辑。
3.用钩子方法给予子类更大的灵活性。
3.最后将方法汇总构成一个不可改变的模版方法。

测试结果

模版方法模式的特点

优点

1.封装性好
2.复用性好
3.屏蔽细节
4.便于维护

缺点

继承，由于Java中是单继承，当使用了模版方法设计模式之后，就不能继承其他类了
如果是重构的项目，很难处理父子类之间的关系

模版方法模式在项目中的应用

适用场景
1.算法或操作遵循相似的逻辑
2.重构时（把相同的代码抽取到父类中）
3.重要、复杂的算法，核心算法设计为模版算法

行业案例

在电信运营商运营的过程中，会出现种类繁多数量巨大的日志，这时候就可以运用到模版方法设计模式。
首先，抽取共性，在获得规律
1.获得文件
2.打开文件
3.读取日志结构
4.处理单行日志
5.清理工作
我们可以发现，不管是什么样的日志，都只有第4步不同，这就可以运用模版设计模式，在在其中添加一些钩子函数，就可以更灵活的运用。
