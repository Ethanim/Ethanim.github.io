三表查询及格和不及格的学生数量

发表于 2018-03-18 | 分类于 SQL

题目介绍

这是我在美团二面遇到的题，应该没记错
学生表，课程表，成绩表，一条sql写出每门课程的及格和不及格的学生数量
下面的SQL以Mysql数据库为准

建表

create table student(
    s_id int auto_increment primary key,
    name varchar(10)
);
create table course(
    c_id int auto_increment primary key,
    c_name varchar(20)
);
create table stu_cou(
    s_id int,
    c_id int,
    score int
);

方法一

因为是求数量，学生表内存放的是学生的名字，所以无用
先分别连表查处及格和不及格的人数，再把两个表连起来

select t.c_name,pass,fail from 
(select c.c_name,count(*) pass from course c join stu_cou sc on c.c_id = sc.c_id
where score>=60 group by c.c_name) t left join 
(select c.c_name,count(*) fail from course c join stu_cou sc on c.c_id = sc.c_id
where score<60 group by c.c_name) t2 on t.c_name = t2.c_name

查询结果

有些问题

此方法有一个问题，如果交换查询及格和不及格语句位置，就会出现如下问题
因为测试数据中，英语都是及格的，左表（别名t表）查询不到不及格的英语，
左连接查询以左表为主，导致没有这条数据
MySQL中没有全外连接，用UNION也会出现些问题，
如果换成Oracle的full join 也有问题
此方法受限，需要左表能查到全部数据

select t.c_name,pass,fail from 
(select c.c_name,count(*) fail from course c join stu_cou sc on c.c_id = sc.c_id
where score<60 group by c.c_name) t left join 
(select c.c_name,count(*) pass from course c join stu_cou sc on c.c_id = sc.c_id
where score>=60 group by c.c_name) t2 on t.c_name = t2.c_name

方法二

用科目名称与(score<60)分组，能分出每门课程及格与不及格的数量
利用if来输出及格和不及格
此方法数据看起来会有些多余，MySQL中可用，Oracle会报错

select c.c_name,count(*),if(sc.score<60,'不及格','及格') score from course c 
join stu_cou sc on c.c_id = sc.c_id
group by  c.c_name,sc.score<60

查询结果

方法三

先只通过stu_cou表来分别查询出及格人数和不及格人数，分别起别名(t,t2),
在把course表，t表与t2表做连接起来，由于课程表是左表，所以不会出现方法一的情况
此方法，MySQL与Oracle通用

select c_name,pass,fail from course c left join 
(select c_id,count(*) pass from stu_cou where score>=60 group by c_id) t
on c.c_id = t.c_id left join
(select c_id,count(*) fail from stu_cou where score<60 group by c_id) t2 
on c.c_id = t2.c_id;
