代理设计模式（Proxy）

发表于 2018-03-24

代理模式概念介绍

概念：为其他对象提供一种代理，以控制对这个对象的访问，代理对象起到了中介的功能，去掉了某些功能，或者增加一些额外的服务

1、什么是代理模式？
代理模式定义是：为其他对象提供一种代理以控制对这个对象的访问。
通俗的讲，代理模式就是：通过一个类访问另一个类的行为就是代理模式，比如通过A类访问B类的行为，那么A类就是B的代理类，当然前提是A类和B类都必须实现相同的接口，无论是静态代理还是动态代理，动态代理没有明显的实现，它是在运行时动态实现。

为什么必须要实现相同的接口呢？
因为代理类是要代理另一个类做了一些操作，无论是增强了还是控制了，其最主要的行为并没有发生变化，只是代理做一些操作，不是要更改被代理类的行为。所以如果不实现相同的接口，只是单纯的说“通过一个类访问另一个类的行为就是代理模式”，那么就会可能出现代理类与被代理类表现出不一致的行为，比如代理类的一个行为叫login，表示登录的行为，而login这个行为中调用gameUpgrade游戏升级的行为，就会导致在高层模块调用了登录这个行为，而实际表现了游戏升级的行为这样一个严重错误的逻辑情况。而实现相同的接口，共同的行为（多态），就可以避免这种情况。

代理模式概念及分类

  • 远程代理：为不同地理的对象，提供局域网代表对象，例：通过远程代理可以监控各个店铺，是指能直观的了解店内信息
  • 虚拟代理：根据需要将资源消耗很大的对象进行延迟，真正需要的时候进行创建，例：网页加载的图片太大，可先用一张虚拟图片代替。
  • 保护代理：控制用户的访问权限
  • 只能引用代理：提供对目标对象额外服务

了解 JDK 动态代理

动态产生代理，实现对不同类，不同方法的代理

Java动态代理类位于 java.lang.reflect 包下，一般主要涉及到一下两个类：
(1)Interface InvocationHandler：该接口中仅定义了一个方法
public object invoke(Object obj, Method method, Object[] args)
在实际使用时，第一个参数obj一般是指代理类，method 是被代理的方法，args 为该方法的参数数组。这个抽象方法在代理类中动态实现。
(2)Proxy：该类即为动态代理类
static Object newProxyInstance(ClassLoader loader, Class[] interfaces, InvocationHandler h)：返回代理类的一个实例，返回后的代理类可以当作被代理类使用（可使用被代理类的在接口中声明过的方法）

所谓 Dynamic Proxy 是这样一种 class：
它是在运行时生成的 class
该 class 需要实现一组 interface
使用动态代理类时，必须实现 InvocationHandler 接口

动态代理实现步骤
1.创建一个实现接口 InvocationHandler 的类，它必须实现 invoke 方法
2.创建被代理的类以及接口
3.调用 Proxy 的静态方法，创建一个代理类
newProxyInstance(ClassLoader loader, Class[] interfaces, InvocationHandler h)
4.通过代理调用方法

区别
JDK动态代理
1.只能代理实现了接口的类
2.没有接口的类不能实现JDK的动态代理
CGLIB动态代理
1.针对类来实现代理的
2.对指定目标类产生一个子类，通过方法拦截技术拦截所有父类方法的调用。
